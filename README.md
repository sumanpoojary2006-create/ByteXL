# ByteXL Workspace

Python curriculum authoring workspace. Organized into four top-level areas:

## `content/` — the deliverables
- `Semester 1/` — Units 1–13 (reading material, per-unit folders)
- `Semester 2/` — Units 1–14
- `Question Bank/` — Coding Questions, MCQ, Review, Template
- `Curriculum/` — the two curriculum workbooks (`.xlsx`)

## `tools/` — apps and scripts
- `scripts/` — Python authoring/upload scripts (formerly "py files")
- `image-converter/` — Flask/Vercel image-to-URL app
- `image-converter-extension/` — companion Chrome extension
- `vercel-coding-question-uploader/` — Vercel app
- `vercel-onecompiler-builder/` — Vercel app

## `docs/`
- `ByteXL_Uploader_Technical_Walkthrough.docx`
- `CODING_QUESTION_UPLOAD_AUTOMATION.md`

## `archive/` — backups and regenerable build artifacts
- `Semester 1.zip` (391 MB backup)
- `api_probe/`, `curriculum_build/`, `onecompiler_build/`, `onecompiler_smoke/`
  (former hidden `.` build/cache dirs; regenerated by the scripts when run)

---

### Running the scripts

Scripts in `tools/scripts/` resolve content via a `CONTENT_ROOT` that points to
`../../content` (i.e. this folder's `content/`), so they can be run from anywhere.
They were relocated from the workspace root, and their paths were updated to match
this layout. The onecompiler/upload scripts still require a `bytexl_config.json`
(API token) and the `onecompiler_*.html` companion files, which are not present in
this workspace — supply those before running the upload pipeline.
